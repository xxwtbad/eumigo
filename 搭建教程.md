# 博客系统——从零到一的部署指南

## 前言

欢迎来到 ddmer-1 博客系统！这是一个基于 Next.js 14 + Prisma + PostgreSQL + Cloudflare R2 的现代个人博客系统，支持文章、相册、图书、音乐等多种内容形式，以及 GitHub OAuth 登录和评论系统。

本教程将带你从零开始，一步步完成整个系统的部署，即使你是技术新手也能轻松上手。

## 一、准备工作

### 1.1 所需账号

在开始之前，请准备好以下账号（均为免费或免费套餐）：

| 序号 | 账号 | 用途 | 注册地址 |
|------|------|------|----------|
| 1 | GitHub | 代码托管和 OAuth 登录 | https://github.com |
| 2 | Vercel | 网站部署托管 | https://vercel.com |
| 3 | Neon | PostgreSQL 数据库 | https://console.neon.tech |
| 4 | Cloudflare | R2 对象存储 + 可选 CDN | https://dash.cloudflare.com |
| 5 | 自定义域名 | 可选，用于绑定域名 | https://my.dnshe.com/ 或其他域名商 |

### 1.2 所需工具
- https://www.paypal.com paypal（国行）
- 在线随机字符串生成器：https://www.xyutil.com/random-string/zh/

### 1.3 环境变量清单

以下是部署时需要配置的环境变量，请提前准备好：

| 变量名 | 说明 | 是否必须 | 来源 |
|--------|------|----------|------|
| DATABASE_URL | Neon 数据库连接串 | ✅ 必须 | Neon 控制台 |
| SECRET_KEY | JWT 签名密钥（64位随机字符串） | ✅ 必须 | 自己生成 |
| FRONTEND_ORIGIN | 前端域名（末尾不要 /） | ✅ 必须 | Vercel 给的域名 |
| NEXT_PUBLIC_API_URL | 前端 API 地址（可选） | 可选 | 和 FRONTEND_ORIGIN 相同 |
| GITHUB_CLIENT_ID | GitHub OAuth App ID | ✅ 必须 | GitHub |
| GITHUB_CLIENT_SECRET | GitHub OAuth App Secret | ✅ 必须 | GitHub |
| ADMIN_GITHUB_USERS | 允许登录后台的 GitHub 用户名 | ✅ 必须 | 你的 GitHub 用户名 |
| R2_ACCOUNT_ID | Cloudflare 账户 ID | ✅ 必须 | Cloudflare |
| R2_ACCESS_KEY_ID | R2 API Access Key | ✅ 必须 | Cloudflare |
| R2_SECRET_ACCESS_KEY | R2 API Secret Key | ✅ 必须 | Cloudflare |
| R2_BUCKET_NAME | 存储桶名 | ✅ 必须 | Cloudflare |
| R2_PUBLIC_URL | 文件公共访问域名 | 强烈建议 | Cloudflare |

## 三、第二步：创建数据库（Neon）

Neon 提供免费的 PostgreSQL 数据库，非常适合个人项目。

### 3.1 创建项目

1. 打开 https://console.neon.tech，登录
2. 点击页面上的 **Create a project**（创建项目）
3. 在弹出的页面里填写：
   - **Project name**：`ddmer-db`
   - **Region**（地区）：**Singapore**（新加坡，离中国最近）
   - 其他保持默认，点击 **Create project**

### 3.2 获取连接字符串

1. 创建成功后，页面会显示一个 **Connection string**（连接字符串），格式如下：

   ```
   postgresql://用户名:密码@ep-xxx.us-east-1.aws.neon.tech/neondb?sslmode=require
   ```

2. 点击旁边的 **复制按钮**，把这一串保存到记事本
3. 这就是 `DATABASE_URL` 的值

> **提示**：如果 Neon 数据库名不是 `neondb`，请在连接字符串末尾把 `neondb` 改成你的实际数据库名。

---

## 四、第三步：创建文件存储（Cloudflare R2）

R2 是 Cloudflare 提供的对象存储服务，免费额度 10GB，用来存放上传的图片、文件等。

### 4.1 创建存储桶（Bucket）

1. 打开 https://dash.cloudflare.com，登录
2. 左侧菜单找到 **R2 Object Storage**，点进去
3. 点 **Create bucket**（创建存储桶）
4. **Bucket name** 填：`ddmer-files`（可自定义）
5. 点 **Create**

### 4.2 配置公共访问

1. 进入刚创建的 bucket，点顶部 **Settings** → **Public access**
2. 点 **Connect domain**，绑定一个自己的域名，比如 `files.yourdomain.com`
   - 如果没有自己的域名，可以用 Cloudflare 提供的临时域名
3. 复制这个域名，前面加上 `https://`，保存到记事本：
   - 例如：`https://files.yourdomain.com`
   - 这就是 `R2_PUBLIC_URL`

### 4.3 创建 API 密钥

1. 还是在 Cloudflare R2 页面，左侧菜单点 **Manage R2 API Tokens**
2. 点 **Create API token**
3. **Permissions**（权限）选 **Object Read & Write**
4. **Specify bucket**（指定存储桶）选刚才创建的 `ddmer-files`
5. 点 **Create**

创建成功后，页面会显示三个值，全部保存到记事本：

| 值 | 对应环境变量 |
|----|-------------|
| Account ID | `R2_ACCOUNT_ID` |
| Access Key ID | `R2_ACCESS_KEY_ID` |
| Secret Access Key | `R2_SECRET_ACCESS_KEY` |

> **注意**：Secret Access Key 只显示一次，一定要保存好！

---

## 五、第四步：创建 GitHub OAuth 应用

GitHub OAuth 用于博客的评论系统和后台登录。

**重要**：每个部署者必须创建自己的 GitHub OAuth App，不能共用同一个。

### 5.1 创建应用

1. 打开 https://github.com/settings/developers
2. 点 **OAuth Apps** 标签
3. 点 **New OAuth App**（新建 OAuth 应用）
4. 填写：
   - **Application name**：`ddmer-1-blog`
   - **Homepage URL**：`https://你的域名.vercel.app`
   - **Authorization callback URL**：`https://你的域名.vercel.app/api/auth/github/callback`
5. 点 **Register application**

### 5.2 获取密钥

1. 页面会显示 **Client ID**，复制下来，这是 `GITHUB_CLIENT_ID`
2. 点 **Generate a new client secret**
3. 复制新生成的 **Client Secret**，这是 `GITHUB_CLIENT_SECRET`

> **注意**：Client Secret 只显示一次，一定要保存好！

---

## 六、第五步：在 Vercel 上部署

### 6.1 创建项目

1. 打开 https://vercel.com，登录
2. 点 **Add New → Project**
3. 在列表中找到你 Fork 的仓库，点 **Import**
4. 进入项目配置页：
   - **Framework Preset**：自动识别为 Next.js，不用改
   - **Build Command**：保持默认（`npm run build`）
   - **Install Command**：保持默认（`npm install`）

### 6.2 配置环境变量

点击 **Environment Variables**（环境变量），逐条添加：

| 变量名 | 值（示例） | 来源 |
|--------|-----------|------|
| `DATABASE_URL` | `postgresql://...` | 第三步 Neon 复制的连接串 |
| `SECRET_KEY` | `a3f8e2b...`（64位随机字符串） | 用在线工具生成 |
| `FRONTEND_ORIGIN` | `https://你的域名.vercel.app` | Vercel 给的域名 |
| `NEXT_PUBLIC_API_URL` | `https://你的域名.vercel.app` | 和 FRONTEND_ORIGIN 相同 |
| `GITHUB_CLIENT_ID` | `Iv23li...` | 第五步 GitHub 复制的 |
| `GITHUB_CLIENT_SECRET` | `xxx...` | 第五步 GitHub 复制的 |
| `ADMIN_GITHUB_USERS` | `你的GitHub用户名` | 允许登录后台的用户名 |
| `R2_ACCOUNT_ID` | `xxx...` | 第四步 Cloudflare 复制的 |
| `R2_ACCESS_KEY_ID` | `xxx...` | 第四步 Cloudflare 复制的 |
| `R2_SECRET_ACCESS_KEY` | `xxx...` | 第四步 Cloudflare 复制的 |
| `R2_BUCKET_NAME` | `ddmer-files` | 第四步创建的 bucket 名 |
| `R2_PUBLIC_URL` | `https://files...` | 第四步配置的公共域名 |

> **注意**：
> - `FRONTEND_ORIGIN` 末尾不要加 `/`
> - `SECRET_KEY` 请使用至少 32 位的随机字符串，可以在 https://www.xyutil.com/random-string/zh/ 生成
> - `ADMIN_GITHUB_USERS` 填写你的 GitHub 用户名，多个用户名用英文逗号分隔

### 6.3 部署

点 **Deploy**（部署），等待 1-2 分钟完成构建。

## 七、第六步：初始化数据库（最关键！）

Vercel 部署完成后，数据库还是空的，需要导入表结构。

### 7.1 操作步骤

1. 打开 https://console.neon.tech，登录
2. 点击你的项目名（如 `ddmer-db`）
3. 在左侧菜单找到 **SQL Editor**，点击进入
4. 打开本仓库中的 `init.sql` 文件，**全选复制**其内容
5. 在 SQL Editor 的输入框里，按 `Ctrl + V` 粘贴
6. 点击上方的 **Run** 按钮执行
7. 等待几秒，看到 **Success** 提示就成功了

> **提示**：`init.sql` 使用了 `IF NOT EXISTS` 和 `ON CONFLICT DO NOTHING`，可以安全重复执行，不会覆盖已有数据。

### 7.2 默认管理员账户

数据库初始化完成后，系统会自动创建一个默认管理员账户：

| 项目 | 值 |
|------|-----|
| 用户名 | `admin` |
| 密码 | `admin123` |

> **安全提示**：首次登录后，请立即在后台修改默认密码！

你也可以使用 GitHub OAuth 登录后台（前提是你的 GitHub 用户名已添加到 `ADMIN_GITHUB_USERS` 环境变量中）。

## 九、常见问题

### Q1：部署后页面报错 500？

检查 Vercel 的 **Logs**（日志），最常见的原因是环境变量配置错误，特别是：
- `DATABASE_URL` 中的数据库名是否正确
- `FRONTEND_ORIGIN` 末尾是否多了 `/`
- `SECRET_KEY` 是否已设置

### Q2：GitHub 登录失败？

1. 检查 `GITHUB_CLIENT_ID` 和 `GITHUB_CLIENT_SECRET` 是否正确
2. 检查 GitHub OAuth App 的 **Authorization callback URL** 是否和你的域名完全一致
3. **注意**：每个人的 OAuth App 只能用于自己的域名，不能共用

### Q3：图片上传失败？

1. 检查 R2 相关的四个环境变量是否都已正确配置
2. 确认 R2 存储桶已开启公共访问
3. 确认 `R2_PUBLIC_URL` 已正确配置

### Q4：数据库连接失败？

1. 确认 `DATABASE_URL` 使用的是 Neon 的 **Pooled connection string**（带 `-pooler` 的地址）
2. 确认连接字符串中的数据库名正确（默认为 `neondb`）

### Q5：init.sql 执行报错 `relation already exists`？

说明你之前已经执行过一次。新的 `init.sql` 使用了 `IF NOT EXISTS`，可以安全重复执行。如果你使用的是旧版本 SQL，请使用最新仓库中的 `init.sql`。

### Q6：收藏夹页面报错 500？

这是因为数据库中还没有创建 `BookmarkCategory` 和 `BookmarkSite` 表。请确保已完整执行 `init.sql`。

请使用仓库根目录下的 `init.sql` 文件，在 Neon 控制台的 SQL Editor 中执行即可。

> **特性**：

> - 已存在的表和数据不会被覆盖
